# Push_swap 課題まとめ

> *Because Swap_push doesn't feel as natural*
> バージョン: 1.1（グループ課題：2人で取り組む）

---

## 概要

整数の入ったスタックを、限られた操作セットで、できるだけ少ない手数でソートする。
`push_swap` という名前のCプログラムを作り、スタックaをソートするための最小のPush_swap操作列を標準出力に表示する。

---

## 第II章：共通ルール

- プロジェクトはC言語で書くこと
- **Norm**（コーディング規約）に従うこと。ボーナスファイルも含めてNormチェックされ、エラーがあれば0点
- 関数が予期せず終了してはならない（セグフォ、バスエラー、double free等）→ 0点扱い

- Makefileを提出する場合は `-Wall -Wextra -Werror` フラグ付きで `cc` でコンパイルすること。不必要な再リンクをしないこと
- Makefileには少なくとも `$(NAME)`, `all`, `clean`, `fclean`, `re` ルールを含めること
- ボーナスを提出する場合はMakefileに `bonus` ルールを追加し、ボーナスは `_bonus.{c/h}` ファイルに置くこと
- libftを使う場合は `libft` フォルダにソースとMakefileをコピーし、プロジェクトのMakefileでlibftをビルドしてからプロジェクトをビルドすること
- テストプログラムを作ることを推奨する（提出不要・採点対象外）

## 第IV章：はじめに

**Push swap** プロジェクトは非常にシンプルなアルゴリズム課題：データをソートせよ。

手元には整数の集合、2つのスタック、両スタックを操作する命令セットがある。
目標は `push_swap` というCプログラムを書き、引数として受け取った整数を *Push swap言語* の操作でソートする最小の命令列を標準出力に表示すること。

---

## 第V章：目的

このプロジェクトの目的は、**アルゴリズムの計算量**を具体的に体験すること。

数をソートするのは簡単だが、2つのスタックと限られた操作だけで**速く**ソートするのは全く別の話。完全にランダムなリストとほぼソート済みのリストでは、アルゴリズムの選択が結果に大きく影響する。

### VI.2 ルール

- **スタック `a`** と **スタック `b`** の2つのスタックを使う
- 開始時：
  - スタック `a` には重複なしのランダムな整数（正負混在可）が入っている
  - スタック `b` は空
- 目標：スタック `a` を昇順にソートすること

#### 使用可能な操作

| 命令 | 説明 |
|------|------|
| `sa` (swap a) | スタック `a` の先頭2要素を入れ替える。要素が1つ以下なら何もしない |
| `sb` (swap b) | スタック `b` の先頭2要素を入れ替える。要素が1つ以下なら何もしない |
| `ss` | `sa` と `sb` を同時に実行 |
| `pa` (push a) | スタック `b` の先頭要素をスタック `a` の先頭に移す。`b` が空なら何もしない |
| `pb` (push b) | スタック `a` の先頭要素をスタック `b` の先頭に移す。`a` が空なら何もしない |
| `ra` (rotate a) | スタック `a` の全要素を1つ上にずらす。先頭要素が末尾になる |
| `rb` (rotate b) | スタック `b` の全要素を1つ上にずらす。先頭要素が末尾になる |
| `rr` | `ra` と `rb` を同時に実行 |
| `rra` (reverse rotate a) | スタック `a` の全要素を1つ下にずらす。末尾要素が先頭になる |
| `rrb` (reverse rotate b) | スタック `b` の全要素を1つ下にずらす。末尾要素が先頭になる |
| `rrr` | `rra` と `rrb` を同時に実行 |

### VI.3 アルゴリズム要件

アルゴリズムの計算量（時間・空間）を厳密に理解させるため、**4つの異なるソート戦略**を実装し、`push_swap` プログラムに組み込む必要がある。プログラムは入力の状態に応じてランタイムに戦略を選択できること。

#### VI.3.1 計算量モデルと操作制約

全戦略はCで実装し、Push_swap操作列を生成してソートを行う。

- Cアルゴリズムが入力を解析し、適切な操作列（`sa`, `sb`, `ss`, `pa`, `pb`, `ra`, `rb`, `rr`, `rra`, `rrb`, `rrr`）を生成する
- 戦略の出力は、スタックをソートする操作列そのもの
- 計算量クラスを主張する場合、それは古典的な配列ベースのアルゴリズムの理論計算量ではなく、**生成されるPush_swap操作数**で測定されたコストを反映すること

#### VI.3.2 乱れ度（disorder）※必須

「乱れ度（disorder）」は 0〜1 の数値で、初期スタック `a` がどれだけソートされていないかを表す。

- 既にソート済み → disorder = 0
- 最悪の並び → disorder = 1
- 中間 → スタックが部分的にソートされているが乱れている

**計算方法：** スタック内の全ペアを調べ、大きい数が小さい数の前にある「逆順ペア」の数を数える。逆順ペアが多いほど disorder は 1 に近づく。

```
function compute_disorder(stack a):
    mistakes = 0
    total_pairs = 0
    for i from 0 to size(a)-1:
        for j from i+1 to size(a)-1:
            total_pairs += 1
            if a[i] > a[j]:
                mistakes += 1
    return mistakes / total_pairs
```

> **重要：disorder は操作を行う前に計測すること**

#### VI.3.3 必須戦略

4つの戦略をすべて実装すること：

**1. シンプルアルゴリズム（O(n²)）**
O(n²) クラスの基本アルゴリズムを少なくとも1つ実装する。例：
- 挿入ソートの適応版
- 選択ソートの適応版
- バブルソートの適応版
- シンプルな最小値/最大値抽出法

**2. 中程度アルゴリズム（O(n√n)）**
O(n√n) クラスのアルゴリズムを少なくとも1つ実装する。例：
- チャンク分割ソート（√n 個のチャンクに分割）
- ブロックベースのパーティション法
- √n 個のバケットを使ったバケットソート適応版
- 範囲ベースのソート戦略

**3. 複雑アルゴリズム（O(n log n)）**
O(n log n) クラスのアルゴリズムを少なくとも1つ実装する。例：
- 基数ソート適応版（LSDまたはMSD）
- 2スタックを使ったマージソート適応版
- スタックパーティションを使ったクイックソート適応版
- ヒープソート適応版
- Binary Indexed Tree アプローチ

**4. カスタム適応アルゴリズム（学習者設計）**
計測した disorder に応じて異なる内部手法を選択する**適応型**戦略を設計する。使う内部アルゴリズムは自由だが、各レジームで以下の**計算量目標**を満たすこと（Push_swap操作モデルで）：

| 乱れ度 | 計算量目標 |
|--------|-----------|
| disorder < 0.2（低乱れ） | O(n²) |
| 0.2 ≤ disorder < 0.5（中乱れ） | O(n√n) |
| disorder ≥ 0.5（高乱れ） | O(n log n) |

閾値の根拠・各レジームで使う内部手法・時間と空間の計算量の上界をREADME.md等に文書化すること。

### VI.5 push_swap プログラム

| 項目 | 内容 |
|------|------|
| プログラム名 | `push_swap` |
| 提出ファイル | `Makefile`, `*.h`, `*.c` |
| Makefile | `NAME`, `all`, `clean`, `fclean`, `re` |
| 引数 | スタック a：整数のリスト |
| 使用可能な外部関数 | `read`, `write`, `malloc`, `free`, `exit`、自作の `ft_printf` 相当関数 |
| libft使用 | 可 |
| 説明 | スタックをソートする |

#### プログラムの仕様

- **Makefile**を提出すること。不必要な再リンクをしないこと
- **グローバル変数は禁止**
- `push_swap` プログラムは以下の引数を受け取る：
  - スタック `a`（整数のリスト。最初の引数がスタックの先頭）
  - オプションの**戦略セレクタ**：
    - `--simple`：O(n²) アルゴリズムを強制使用
    - `--medium`：O(n√n) アルゴリズムを強制使用
    - `--complex`：O(n log n) アルゴリズムを強制使用
    - `--adaptive`：disorder に基づく適応アルゴリズムを強制使用（**セレクタ未指定時のデフォルト**）
- スタック `a` をソートするための最小のPush_swap操作列を表示すること（最小値が先頭）
- 操作は `\n` で区切ること（それ以外の文字は不可）
- 各アルゴリズムで主張する計算量クラスは、このモデルで有効であること
- 戦略セレクションは**すべての有効な入力**で機能すること。入力サイズや乱れ度に関わらず動作すること
- 引数が指定されない場合、何も表示せずにプロンプトを返すこと
- エラー時は標準エラー出力に `"Error"` + `\n` を表示する
  - エラー例：整数でない引数、有効範囲外の整数、重複値
- バイナリには4つの戦略（Simple O(n²)、Medium O(n√n)、Complex O(n log n)、Adaptive）をすべて組み込むこと。`--bench` モードで選択された戦略名と計算量クラスが確認できること

#### ベンチマークモード（`--bench`）

`--bench` フラグを付けると、ソート後に以下を **stderr** に表示する（フラグがない場合は表示しない）：

- 計算した `disorder`（小数点以下2桁の%表示）
- 使用した戦略名とその理論計算量クラス
- 操作の総数
- 各操作の種類ごとの数（`sa`, `sb`, `ss`, `pa`, `pb`, `ra`, `rb`, `rr`, `rra`, `rrb`, `rrr`）

### 注意事項

- 完璧とは：必須パートが完全に実装され、エラーなく機能し、**すべてのベンチマークを例外なく通過**すること

### VIII.1 checker プログラム

| 項目 | 内容 |
|------|------|
| プログラム名 | `checker` |
| 提出ファイル | `*.h`, `*.c` |
| Makefile | `bonus` |
| 引数 | スタック a：整数のリスト |
| 使用可能な外部関数 | `read`, `write`, `malloc`, `free`, `exit`、自作の `ft_printf` 相当関数 |
| libft使用 | 可 |
| 説明 | ソート操作を実行して検証する |

#### checker の仕様

- 引数としてスタック `a`（整数のリスト）を受け取る（最初の引数がスタックの先頭）
- 引数なし → 何も表示せずに終了
- 標準入力から操作を読み込む（各命令は `\n` で終わる）
- 全命令を読み込んだ後、受け取ったスタック `a` に対して実行する
- スタック `a` がソート済みかつスタック `b` が空 → 標準出力に `"OK"` + `\n` を表示
- それ以外 → 標準出力に `"KO"` + `\n` を表示
- エラー時 → **標準エラー出力**に `"Error"` + `\n` を表示
  - エラー例：整数でない引数、範囲外の整数、重複値、存在しない命令、不正なフォーマットの命令

```bash
$> ./checker 3 2 1 0
rra
pb
sa
rra
pa
OK

$> ./checker 3 2 1 0
sa
rra
pb
KO

$> ./checker 3 2 one 0
Error

$> ./checker "" 1
Error
```

> **注意：** 提供されているバイナリと完全に同じ動作をしなくてよい。エラー管理は必須だが、引数のパース方法は自由に決めてよい。
